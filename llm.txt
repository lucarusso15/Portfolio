# Portfolio — LLM Reference

This document is a complete technical reference for AI assistants continuing work on this project.
It describes the architecture, all configuration options, the rendering pipeline, CSS structure,
and responsive behaviour in full detail.

---

## What this project is

A minimal, fully static photography portfolio website.
- No server-side language (no PHP, Node, Python, Ruby, etc.).
- No build tools (no Webpack, Vite, npm, etc.).
- No frameworks (no React, Vue, etc.).
- No databases.
- Pure HTML + CSS + JavaScript (vanilla ES6).
- Works on any static file host with zero configuration.

---

## How pages work

Each HTML page (index.html, series-2.html, etc.) follows this pattern:

```html
<head>
  <link rel="icon" id="site-favicon" href="images/favicon.png">
  <link rel="stylesheet" href="css/style.css">
</head>
<body>
  <header data-header></header>
  <div class="page-body" id="page-body"></div>
  <script src="config.js"></script>
  <script src="js/portfolio.js"></script>
  <script>
    document.addEventListener('DOMContentLoaded', () => {
      const serie = PORTFOLIO_CONFIG.series[0]; // index varies by page
      document.title = PORTFOLIO_CONFIG.photographer.name;
      buildHeader(serie.id);
      document.getElementById('page-body').innerHTML = buildSerie(serie);
    });
  </script>
</body>
```

Key points:
- `config.js` is loaded synchronously before `portfolio.js`.
- `portfolio.js` registers a `DOMContentLoaded` listener that calls `applyLayoutSettings()`.
- The page's own `DOMContentLoaded` listener fires after portfolio.js's listener (registered order).
- `applyLayoutSettings()` sets CSS custom properties on `document.documentElement.style` (inline style on `<html>`).
- The page's listener calls `buildHeader(id)` and `buildSerie(serie)` to generate HTML.
- CSS variables are set before HTML is generated, so layout is correct on first render.

## config.js — full structure

```js
const PORTFOLIO_CONFIG = {

  photographer: {
    name:    string,       // Displayed in navbar and browser title
    city:    string,       // Displayed on Info page (optional)
    favicon: string,       // Path to favicon image, e.g. "images/favicon.png"
    bio:     string,       // Multi-paragraph text — separate with blank lines

    email:   string,       // Contact email — rendered as mailto: link. "" to hide.

    links: [               // Array of social / website links shown on Info page
      {
        label:       string,  // For config readability only — NOT displayed on page
        displayLink: string,  // The text rendered as the clickable link
        link:        string,  // Full URL — must start with https://
      },
      // ...
    ],
  },

  layout: {
    contentTopMargin: number,  // px — space between navbar and content, desktop only
    pageSidePadding:  number,  // px — left/right padding on all pages
    textColumnWidth:  number,  // px — width of the left text column
    textMarginRight:  number,  // px — gap between text column and photos
    textMarginBottom: number,  // px — extra space below the text block

    fontSize: {
      navLinks:    number,  // px
      seriesTitle: number,  // px
      bodyText:    number,  // px
      caption:     number,  // px
      infoLinks:   number,  // px — links on Info page
      infoQuote:   number,  // px — random phrase on Info page
    },
  },

  defaults: {
    photoHeight: number,  // px — height for horizontal layout photos
    photoWidth:  number,  // px — default width for vertical layout photos
    photoGap:    number,  // px — gap between photos (all layouts)
    marginTop:    number, // px — default top margin per photo
    marginBottom: number, // px — default bottom margin per photo
    marginLeft:   number, // px — default left margin per photo
    marginRight:  number, // px — default right margin per photo
  },

  quotes: [string, ...],  // Random phrase picked at page load on the Info page

  series: [ SeriesObject, ... ],  // First entry = index.html home page
};
```

## SeriesObject format

```js
{
  id:     string,    // Must match the HTML file base name: "series-2" → series-2.html
  title:  string,    // Displayed in navbar and as series heading
  layout: "horizontal" | "vertical" | "grid",
  intro:  string,    // Multi-paragraph intro text — separate with blank lines

  // For horizontal and vertical layouts:
  photos: [ PhotoObject, ... ],

  // For grid layout only:
  rows:   [ [ PhotoObject, ... ], ... ],
}
```

---

## PhotoObject — per-photo overrides

All fields except `src` are optional. Unset fields fall back to `defaults`.

```js
{
  src:         string,   // Required — path to image file
  caption:     string,   // Optional — shown below photo. "" to hide.
  h:           number,   // Override height in px
  w:           number,   // Width as fraction 0–1 (vertical and grid layouts)
  marginTop:   number,   // px
  marginBottom: number,  // px
  marginLeft:  number,   // px
  marginRight: number,   // px
}
```

## Layout types — rendering logic

### Horizontal (`buildHorizontalSerie`)

Desktop/laptop:
- `<div class="series-row">` — flex row, `overflow-x: auto`, `scrollbar-width: none`
- Left: `<div class="text-col">` — fixed width from `--text-col-w`
- Right: `<div class="photos-horizontal">` — flex row, `flex-wrap: nowrap`, photos scroll
- Each photo: `<div class="photo-wrap">` → `<img style="height:Xpx">` where X = CSS var `--photo-h`
- If `p.h` is set, the img gets `style="height:Xpx"` as inline style; otherwise height comes from CSS var

Touch portrait:
- CSS `(orientation: portrait) and (pointer: coarse)` fires
- `.series-row { flex-direction: column }` — text above, photos stack
- `.photos-horizontal { flex-direction: column }` — vertical stack
- Images: `width: 100% !important; height: auto !important` — fill width

Touch landscape:
- JS `_applyViewportProps()` sets `--photo-h` = `min(config.photoHeight, window.innerHeight - 42 - 10 - 10)`
- JS sets `--content-top-margin: 10px` (instead of configured 150px+)
- JS adds class `touch-landscape` to `<html>` element
- CSS `(orientation: landscape) and (pointer: coarse)` fires AND `html.touch-landscape` class applies
- `.page-body { padding-bottom: 0 }` — prevents vertical overflow from bottom padding
- `max-height: calc(100svh - var(--header-h) - var(--content-top-margin) - 10px)` on images — CSS failsafe

### Vertical (`buildVerticalSerie`)

- `<div class="series-vertical-wrap">` — flex row (desktop) / column (touch portrait)
- Left: `<div class="text-col">`
- Right: `<div class="photos-vertical">` — flex column
- Per photo: `<div class="photo-wrap vertical-photo" style="width:Xpx|X%">` → `<img style="width:...; height:...">`
- If `p.w` is set: outer div width = `p.w * 100` %; else width = `defaults.photoWidth`px with `max-width:100%`
- If `p.h` is set: img `height:Xpx; object-fit:cover`; else `height:auto`

Touch landscape: CSS overrides set `width: 100% !important` on each vertical-photo.

### Grid (`buildGridSerie` + `buildGridRow`)

- `<div class="series-grid-wrap">` → `<div class="text-col-grid">` + `<div class="photos-grid">`
- `photos-grid` is a flex column of `<div class="grid-row">` elements
- Each `grid-row` is a flex row of `<div class="photo-wrap grid-photo">` elements

Four photo modes determined by presence of `h` and `w`:
1. `h` only → `flex: 0 0 auto` on outer; img `height:Hpx; width:auto`
2. `w` only → `flex: W W 0` on outer; img `width:100%; height:auto`
3. `h + w` → `flex: W W 0` on outer; img `height:Hpx; width:100%; object-fit:cover`
4. neither → equal share flex on outer; img `width:100%; height:auto`

`SCALE = 100` is used as flex factor multiplier for fractional w values.
Photos without `w` share remaining space after subtracting all explicit `w` values.

Touch landscape: `.series-grid-wrap { flex-direction: column }`, `.grid-photo { flex: 1 1 0 !important }` — all photos in a row scale equally to fill width.
Touch portrait: `.series-grid-wrap { flex-direction: column }`, `.grid-row { flex-wrap: wrap }`, `.grid-photo { flex: 0 0 calc(50% - gap/2) !important }` — 2 per row.
Phone portrait (≤ 640px): `.grid-photo { flex: 0 0 100% !important }` — 1 per row.

## JS — applyLayoutSettings() and _applyViewportProps()

`applyLayoutSettings()` is called once at `DOMContentLoaded` (registered in portfolio.js, fires before page listener).

It reads `PORTFOLIO_CONFIG.layout` and `PORTFOLIO_CONFIG.defaults` and sets CSS custom properties via `document.documentElement.style.setProperty(...)` (inline style on `<html>`).

CSS variables set by JS:
- `--pad` — from `layout.pageSidePadding`
- `--text-col-w` — from `layout.textColumnWidth`
- `--text-margin-right` — from `layout.textMarginRight`
- `--text-margin-bottom` — from `layout.textMarginBottom`
- `--photo-gap` — from `defaults.photoGap`
- `--fs-nav`, `--fs-title`, `--fs-body`, `--fs-caption`, `--fs-info-links`, `--fs-info-quote` — from `layout.fontSize`
- `--photo-h` — computed by `_applyViewportProps()`
- `--content-top-margin` — computed by `_applyViewportProps()`

`_applyViewportProps()` runs immediately and on every `resize` event:

```js
const isTouch  = navigator.maxTouchPoints > 0;   // all iOS/iPadOS incl. Apple Pencil
const isLandsc = window.innerWidth > window.innerHeight;

if (isTouch && isLandsc) {
  // photo height fills viewport exactly
  const avail = window.innerHeight - 42 /*header*/ - 10 /*margin*/ - 10 /*safety*/;
  --photo-h = min(config.photoHeight, max(avail, 50));
  --content-top-margin = 10px;
  html.classList.add('touch-landscape');
} else if (isTouch) {
  --photo-h = config.photoHeight;  // CSS handles stacking
  --content-top-margin = 10px;
  html.classList.add('touch-portrait');
} else {
  --photo-h = config.photoHeight;
  --content-top-margin = layout.contentTopMargin ?? 150;
  html.classList.remove('touch-landscape', 'touch-portrait');
}
```

Note: `navigator.maxTouchPoints > 0` is used instead of `pointer: coarse` because the latter
fails on iPad with Apple Pencil connected (which switches to `pointer: fine`).

## CSS — variables and cascade

Default values defined in `:root` in style.css:

```css
:root {
  --header-h:           42px;
  --pad:                50px;
  --content-top-margin: 150px;
  --text-col-w:         200px;
  --text-margin-right:  40px;
  --text-margin-bottom: 0px;
  --photo-h:            780px;
  --photo-gap:          18px;
  --fs-nav:             15px;
  --fs-title:           20px;
  --fs-body:            14px;
  --fs-caption:         12px;
  --fs-info-links:      14px;
  --fs-info-quote:      22px;
}
```

JS overrides these via inline style on `<html>`, which has higher specificity than `:root` rules.
CSS media queries that also set these variables use `!important` if they need to override the JS inline style.

---

## CSS — responsive media queries

### `@media (orientation: landscape) and (pointer: coarse)`

Targets touch devices (phones and most tablets) in landscape orientation.

Changes:
- `--pad: 22px`
- `.page-body { padding-bottom: 0 }` — prevents vertical overflow in horizontal scroll layout
- vertical layout: photos fill full width
- grid layout: text above, grid-photo `flex: 1 1 0` to fill screen width

### `html.touch-landscape .page-body { padding-bottom: 0 }`

Outside any media query — catches iPad with Apple Pencil (pointer:fine, bypasses coarse query).
Applied when JS adds `touch-landscape` class to `<html>`.

### `@media (orientation: portrait) and (pointer: coarse)`

Targets touch devices in portrait orientation.

Changes:
- `--pad: 22px`
- horizontal layout: stacks to column, images `width:100%; height:auto`
- vertical layout: text above photos, full width
- grid layout: text above, 2 photos per row with `flex-wrap: wrap`
- about page: `.about-wrap { flex-direction: column }`

### `@media (max-width: 640px) and (orientation: portrait)`

Phone portrait — fine-grained overrides:
- Smaller navbar font and padding
- Grid: 1 photo per row

## CSS — photo height safety

For horizontal layout images, two `max-height` declarations are stacked:

```css
.photos-horizontal .photo-wrap img {
  height: var(--photo-h);
  max-height: calc(100vh  - var(--header-h) - var(--content-top-margin) - 60px); /* fallback */
  max-height: calc(100svh - var(--header-h) - var(--content-top-margin) - 10px); /* modern */
  width: auto;
  max-width: none;
}
```

`svh` (small viewport height, Safari 15.4+, Chrome 108+) is always the viewport height
when all browser UI is visible — the safest ceiling for preventing overflow.
The `vh - 60px` fallback provides a conservative estimate on older browsers where `100vh`
may equal the full screen height ignoring the browser toolbar.

## About (Info) page

Structure in `about.html`:
1. `<div class="about-wrap">` — flex row: bio/links on left, portrait photo on right
2. `<div class="info-quote">` — centered text below the about block

The random quote is injected by JS:
```js
const q = quotes[Math.floor(Math.random() * quotes.length)];
document.getElementById('info-quote').textContent = q;
```

Links are rendered from `c.photographer.links` array:
```js
const linksHTML = (c.photographer.links || [])
  .map(l => `<a href="${l.link}" target="_blank" rel="noopener">${l.displayLink}</a>`)
  .join('');
```

Email is handled separately (mailto: scheme).

Favicon is set by `buildHeader()` which runs on every page:
```js
document.getElementById('site-favicon').href = c.photographer.favicon;
```

## Header

Built by `buildHeader(activePage)` in portfolio.js.
Sets `<header data-header>` innerHTML to site name + nav links.
Nav links are generated from `PORTFOLIO_CONFIG.series` array + the Info page link.
Active page link gets `class="active"` (underline via CSS `border-bottom`).
Also updates the favicon from config.

---

## Adding a series

1. Add an object to `PORTFOLIO_CONFIG.series` in config.js
2. Create a new HTML file (copy series-2.html)
3. In the new file, update the series index: `PORTFOLIO_CONFIG.series[N]`

The series ID must match the HTML filename: `id: "series-4"` → `series-4.html`.
The first series (index 0) always maps to `index.html`.

---

## Browser title

All pages set `document.title = PORTFOLIO_CONFIG.photographer.name`.
No series name or "Info" prefix — always just the photographer name.

## Deployment

The `portfolio/` folder is a self-contained static site.
Upload its contents to any static host. No build step required.
The workflow `Start application` runs `cd portfolio && python3 server.py` for local development.
